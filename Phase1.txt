1️⃣ INTRODUCTION TO RUST
📖 What is it?

Rust is a systems programming language that provides memory safety without garbage collection. It was created by Graydon Hoare at Mozilla and first released in 2015. Rust guarantees memory safety through its ownership system, which is checked at compile time.
🎯 When to Use Rust

    Systems programming: Operating systems, device drivers, embedded systems

    Web backends: High-performance web services and APIs

    Blockchain: Smart contracts, blockchain nodes (Solana, Polkadot)

    Game development: Game engines, performance-critical game logic

    CLI tools: Fast, reliable command-line applications

    WebAssembly: Compiling to WASM for web applications

💡 Why Use Rust

    Memory safety: No null pointer dereferences, no buffer overflows, no use-after-free

    Performance: Zero-cost abstractions, no garbage collector overhead

    Concurrency: Fearless concurrency with compile-time guarantees against data races

    Productivity: Rich type system, excellent tooling, great error messages

    Ecosystem: Growing library ecosystem, active community

Rust's toolchain is managed by rustup, which handles installation of the Rust compiler (rustc), package manager (cargo), and toolchains. Cargo is Rust's build system and package manager.
🎯 When to Set Up

    First installation: When starting Rust development

    Updating: Regularly to get new features and fixes

💡 Why Use rustup and Cargo

3️⃣ VARIABLES AND DATA TYPES
📖 What is it?

Variables in Rust are immutable by default. This is a key design choice that prevents accidental mutation and enables compiler optimizations. Data types are either scalar (single value) or compound (multiple values).
🎯 When to Use

    let: For declaring variables (immutable by default)

    mut: When you need to modify a variable after declaration

    const: For compile-time constants that never change

    Shadowing: When you want to reuse variable names with different types or values

💡 Why Immutable by Default

    Safety: Prevents accidental modification

    Optimization: Compiler can make stronger assumptions

    Clarity: Code is easier to reason about

    Concurrency: Eliminates data races

📝 Complete Methods Reference
Variable Declaration
rust

// Why immutable by default: Prevents accidental mutation
// When: Most variable declarations
fn variable_examples() {
    // Basic declaration (immutable)
    let x = 5;
    // x = 10;  // ❌ Cannot reassign immutable variable
    
    // Mutable variable
    let mut y = 5;
    y = 10;  // ✅ Allowed with mut
    
    // Constant (always immutable)
    const MAX_USERS: u32 = 100;
    // MAX_USERS = 200;  // ❌ Constants cannot be changed
    
    // Multiple declarations
    let (a, b) = (1, 2);
}

Shadowing
rust

// Why shadowing: Reuse variable names without mutation
// When: Transforming values, changing types
fn shadowing_examples() {
    let x = 5;
    let x = x + 1;  // Shadow previous x
    let x = x * 2;  // Shadow again
    println!("x = {}", x);  // 12
    
    // Shadowing allows type change
    let spaces = "   ";
    let spaces = spaces.len();  // &str → usize
    println!("Length: {}", spaces);  // 3
    
    // Shadowing vs mut
    let mut count = 5;
    count = 6;  // Only same type allowed
    
    // Shadowing allows different type
    let count = "six";  // Now count is &str
}

4️⃣ OPERATORS
📖 What is it?

Operators in Rust allow you to perform arithmetic, comparison, logical, and bitwise operations. Rust has the same operators as C/C++ but with some Rust-specific behaviors.
🎯 When to Use

    Arithmetic: For mathematical calculations

    Comparison: For decision-making and sorting

    Logical: For combining boolean conditions

    Bitwise: For low-level operations, flags, embedded systems

💡 Why Operators Matter

    Expressiveness: Write concise, readable code

    Performance: Compile to efficient machine code

    Safety: Overflow checking in debug builds

    Consistency: Familiar syntax from other languages

📝 Complete Methods Reference
Arithmetic Operators
rust

// Why arithmetic: Perform mathematical calculations
// When: Number manipulation, calculations

fn arithmetic_operators() {
    let a = 10;
    let b = 3;
    
    // Basic arithmetic
    let sum = a + b;        // 13 - Addition
    let diff = a - b;       // 7  - Subtraction
    let product = a * b;    // 30 - Multiplication
    let quotient = a / b;   // 3  - Integer division (truncates)
    let remainder = a % b;  // 1  - Modulo/Remainder
    
    // Float arithmetic (use decimal points)
    let x = 10.0;
    let y = 3.0;
    let float_div = x / y;  // 3.3333333333333335
    
    // Type matters - can't mix integers and floats
    // let z = 10 / 3.0;     // ❌ Mixed types
    
    // Compound assignment
    let mut count = 10;
    count += 5;              // count = 15
    count -= 3;              // count = 12
    count *= 2;              // count = 24
    count /= 4;              // count = 6
    count %= 4;              // count = 2
}

Comparison Operators
rust

// Why comparison: Make decisions based on values
// When: Conditional logic, sorting, validation

fn comparison_operators() {
    let a = 5;
    let b = 10;
    
    let equal = a == b;        // false - Equal to
    let not_equal = a != b;    // true  - Not equal to
    let greater = a > b;       // false - Greater than
    let less = a < b;          // true  - Less than
    let greater_equal = a >= b; // false - Greater than or equal
    let less_equal = a <= b;    // true  - Less than or equal
    
    // Can chain comparisons (not like Python)
    // let result = 1 < a < 10;  // ❌ Doesn't work in Rust
    let result = 1 < a && a < 10; // ✅ Correct way
    
    // Comparisons on different types
    let x = 5;
    let y = 5.0;
    // let equal = x == y;      // ❌ Can't compare i32 and f64
    let equal = (x as f64) == y; // ✅ Type conversion
}

// Why logical: Combine boolean conditions
// When: Complex conditional logic

fn logical_operators() {
    let a = true;
    let b = false;

    let or = a || b;       // true  - Logical OR (short-circuiting)

Bitwise Operators
rust

// Why bitwise: Low-level flag manipulation
// When: Systems programming, embedded, permissions, flags

fn bitwise_operators() {
    let a = 0b1100;  // 12 in binary
    let b = 0b1010;  // 10 in binary
    
    let and = a & b;        // 0b1000 (8)   - Bitwise AND
    let or = a | b;         // 0b1110 (14)  - Bitwise OR
    let xor = a ^ b;        // 0b0110 (6)   - Bitwise XOR
    let not = !a;           // 0b...0011    - Bitwise NOT (inverts all bits)
    
    // Shift operators
    let left_shift = a << 2;    // 0b110000 (48) - Left shift
    let right_shift = a >> 2;   // 0b0011 (3)    - Right shift
    
    // Common use: flags and permissions
    const READ: u8 = 0b001;
    const WRITE: u8 = 0b010;
    const EXECUTE: u8 = 0b100;
    
    let permissions = READ | WRITE;  // Combine permissions
    
    if permissions & READ != 0 {
        println!("Can read");
    }
    
    if permissions & WRITE != 0 {
        println!("Can write");
    }
}

🏆 Best Practices
rust

// 1. Use integer division carefully (truncates)
let result = 5 / 2;  // 2, not 2.5

// 2. Use float for decimal division
let result = 5.0 / 2.0;  // 2.5

// 3. Use `%` for remainder (works with negative numbers)
let remainder = -5 % 3;  // -2 (not 1 like in Python)

// 4. Use parentheses for complex expressions
let result = (a + b) * c - (d / e);

// 5. Use short-circuiting for efficient conditions
if expensive_check() && cheap_check() { }  // Both checked
if cheap_check() && expensive_check() { }  // Short-circuits

// 6. Use `checked_*` methods for overflow safety
let result = 255u8.checked_add(1);  // Some(256) or None for overflow
let result = 255u8.saturating_add(1);  // 255 (saturates)
let result = 255u8.wrapping_add(1);  // 0 (wraps around)

5️⃣ CONTROL FLOW
📖 What is it?

Control flow statements determine the order in which code executes. Rust provides if expressions, loops (loop, while, for), and pattern matching.
🎯 When to Use

    if/else: Conditional branching

    loop: Infinite loops with explicit breaks

    while: Condition-based loops

    for: Iterating over collections and ranges

💡 Why Rust's Control Flow is Special

    if is an expression: Returns values

    loop can return values: break value pattern

    for is the most common: Safe iteration over collections

    Pattern matching: Powerful match for control flow

📝 Complete Methods Reference
If Expression
rust

// Why if: Conditional execution
// When: Making decisions based on conditions

fn if_expressions() {
    let number = 10;
    
    // Basic if
    if number > 5 {
        println!("Number is greater than 5");
    }
    
    // if-else
    if number > 10 {
        println!("Number > 10");
    } else {
        println!("Number <= 10");
    }
    
    // if-else if chain
    if number > 100 {
        println!("Large");
    } else if number > 50 {
        println!("Medium");
    } else if number > 10 {
        println!("Small");
    } else {
        println!("Tiny");
    }
    
    // if as an expression (returns value)
    let condition = true;
    let result = if condition { 5 } else { 10 };
    println!("Result: {}", result);
    
    // All branches must return same type
    let value = if true {
        42  // i32
    } else {
        100 // i32 (same type)
    };
    
    // Using if in let binding
    let threshold = if number > 10 { 100 } else { 50 };
}

Loop (Infinite)
rust

// Why loop: Repeat indefinitely until break
// When: Polling, event loops, retry mechanisms

fn loop_examples() {
    // Basic infinite loop
    let mut count = 0;
    loop {
        count += 1;
        if count == 5 {
            break;  // Exit loop
        }
        println!("Count: {}", count);
    }
    
    // Loop with break value (returns value)
    let result = loop {
        let value = get_value();
        if value > 100 {
            break value;  // Returns value from loop
        }
    };
    println!("Result: {}", result);
    
    // Loop labels (for nested loops)
    let mut outer = 0;
    'outer: loop {
        outer += 1;
        let mut inner = 0;
        
        loop {
            inner += 1;
            if inner == 3 {
                break;  // Breaks inner loop only
            }
            if outer == 2 && inner == 2 {
                break 'outer;  // Breaks outer loop
            }
        }
    }
    
    // Continue to skip iteration
    let mut i = 0;
    loop {
        i += 1;
        if i % 2 == 0 {
            continue;  // Skip even numbers
        }
        println!("{}", i);
        if i >= 5 {
            break;
        }
    }
}

For Loop
rust

// Why for: Iterate over collections and ranges
// When: Most common loop for iterating

fn for_examples() {
    // Range (exclusive end)
    for i in 0..5 {
        println!("{}", i);  // 0, 1, 2, 3, 4
    }
    
    // Range (inclusive end)
    for i in 0..=5 {
        println!("{}", i);  // 0, 1, 2, 3, 4, 5
    }
    
    // Reverse range
    for i in (0..5).rev() {
        println!("{}", i);  // 4, 3, 2, 1, 0
    }
    
    // Step by
    for i in (0..10).step_by(2) {
        println!("{}", i);  // 0, 2, 4, 6, 8
    }
    
    // Iterate over array
    let arr = [10, 20, 30, 40];
    for element in arr {
        println!("{}", element);
    }
    
    // Iterate with index
    for (index, value) in arr.iter().enumerate() {
        println!("arr[{}] = {}", index, value);
    }
    
    // Iterate by reference (borrow)
    let vec = vec![1, 2, 3];
    for item in &vec {
        println!("{}", item);
    }
    println!("Vector still usable: {:?}", vec);
    
    // Iterate by mutable reference
    let mut vec = vec![1, 2, 3];
    for item in &mut vec {
        *item *= 2;
    }
    println!("Doubled: {:?}", vec);
    
    // Consume vector (take ownership)
    let vec = vec![1, 2, 3];
    for item in vec {
        println!("{}", item);
    }
    // println!("{:?}", vec);  // ❌ vec moved
}

6️⃣ PATTERN MATCHING (match)
📖 What is it?

match is Rust's most powerful control flow construct. It allows you to compare a value against a series of patterns and execute code based on which pattern matches.
🎯 When to Use

    Exhaustive handling: When all possibilities must be handled

    Enum matching: Primary way to work with enums

    Destructuring: Extract values from complex types

    Fallback: Use _ for default cases

💡 Why Match is Powerful

    Exhaustiveness: Compiler ensures all cases are handled

    Pattern destructuring: Extract values in one step

    Match guards: Add additional conditions

    Value returning: match is an expression

📝 Complete Methods Reference
Basic Match
rust

// Why match: Handle multiple cases exhaustively
// When: Working with enums, options, results

fn basic_match() {
    let number = 3;
    
    match number {
        1 => println!("One"),
        2 => println!("Two"),
        3 => println!("Three"),
        _ => println!("Other number"),  // Catch-all
    }
}

Pattern Types
rust

fn pattern_types() {
    // Matching multiple values
    let number = 2;
    match number {
        1 | 2 => println!("One or Two"),
        3 => println!("Three"),
        _ => println!("Other"),
    }
    
    // Matching ranges
    let score = 85;
    match score {
        90..=100 => println!("Excellent"),
        70..=89 => println!("Good"),
        50..=69 => println!("Average"),
        _ => println!("Fail"),
    }
    
    // Matching with destructuring
    let point = (3, 5);
    match point {
        (0, 0) => println!("Origin"),
        (x, 0) => println!("On X-axis at x = {}", x),
        (0, y) => println!("On Y-axis at y = {}", y),
        (x, y) => println!("Point at ({}, {})", x, y),
    }
    
    // Matching structs
    struct Person { name: String, age: u32 }
    let person = Person { name: "Alice".to_string(), age: 30 };
    
    match person {
        Person { name, age: 30 } => println!("{} is 30", name),
        Person { name, age } if age > 30 => println!("{} is older", name),
        Person { name, .. } => println!("{}", name),
    }
}

Match Guards
rust

// Why guards: Add extra conditions to patterns
// When: Complex matching logic

fn match_guards() {
    let number = 6;
    
    match number {
        n if n % 2 == 0 => println!("Even"),
        n if n % 2 != 0 => println!("Odd"),
        _ => unreachable!(),
    }
    
    // Guards with multiple patterns
    let x = 5;
    let y = 10;
    
    match x {
        1..=10 if y > 5 => println!("x in range, y > 5"),
        1..=10 => println!("x in range, y <= 5"),
        _ => println!("x out of range"),
    }
}

Rust has two string types: String (owned, heap-allocated, mutable) and &str (borrowed, immutable view). Strings are UTF-8 encoded.

    Ownership: Clear distinction between owned and borrowed

struct Rectangle {

    fn area(&self) -> u32 {
        self.width * self.height
    }

    fn scale(&mut self, factor: u32) {
        self.width *= factor;
        self.height *= factor;
    }
    
    // Method taking ownership
    fn into_tuple(self) -> (u32, u32) {
        (self.width, self.height)
    }

    fn new(width: u32, height: u32) -> Self {
        Rectangle { width, height }
    }
    
    fn square(size: u32) -> Self {
        Rectangle {
            width: size,
            height: size,
        }
    }
    
    // Getter (immutable)
    fn width(&self) -> u32 {
        self.width
    }
    
    // Setter (mutable)
    fn set_width(&mut self, width: u32) {
        self.width = width;
    }
}

    // Associated function (constructor)

struct Point {

    y: f64,

        Point { x: 0.0, y: 0.0 }
    }
}

impl Point {
    // Geometry methods
    fn distance_from_origin(&self) -> f64 {
        (self.x * self.x + self.y * self.y).sqrt()
    }
    
    fn distance_to(&self, other: &Point) -> f64 {
        let dx = self.x - other.x;
        let dy = self.y - other.y;
        (dx * dx + dy * dy).sqrt()
    }
}

impl Point {
    // Display methods
    fn format(&self) -> String {
        format!("({}, {})", self.x, self.y)
    }
}

1️⃣2️⃣ ENUMS
📖 What is it?

Enums allow you to define a type that can be one of several variants. Each variant can optionally hold data, making enums extremely powerful for representing state.
🎯 When to Use

    State machines: Represent different states with associated data

    Option types: Values that may be absent

    Error handling: Represent different error types

    Algebraic data types: Powerful data modeling

💡 Why Use Enums

    Expressiveness: Model domain precisely

    Safety: Compiler checks all cases

    Flexibility: Each variant can have different data

    Pattern matching: Natural fit with match

// Why Result: Error handling
// When: Operations that can fail

// Standard library definition
enum Result<T, E> {
    Ok(T),
    Err(E),

fn divide(a: f64, b: f64) -> Result<f64, String> {
    if b == 0.0 {
        Err("Division by zero".to_string())
    } else {
        Ok(a / b)
    }
}

fn result_examples() {
    let result = divide(10.0, 2.0);
    
    match result {
        Ok(value) => println!("Result: {}", value),
        Err(e) => println!("Error: {}", e),
    }
    
    // Common methods
    let value = divide(10.0, 2.0).unwrap_or(0.0);
    let error = divide(10.0, 0.0).unwrap_err();
    
    if let Ok(value) = divide(10.0, 2.0) {
        println!("Success: {}", value);
    }
    
    // Chaining
    let result = divide(10.0, 2.0)
        .map(|x| x * 2.0)
        .map_err(|e| format!("Error: {}", e));
}

            Coin::Quarter => 25,
        }

            Coin::Nickel => "Nickel",
            Coin::Dime => "Dime",
            Coin::Quarter => "Quarter",
        }
    }

    }
}

}
